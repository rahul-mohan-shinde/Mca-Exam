# Q2(a) Built-in Pipes in Angular

Sabse pehle samjho **Pipe hota kya hai?**

Pipe ka kaam:

👉 Data ko format karna aur display karna.

Example:

Name:

```text
rahul
```

Pipe lagaya:

```text
RAHUL
```

Data change nahi hua, sirf display format change hua.

---

## Common Built-in Pipes

### 1. UpperCasePipe

```html
{{ name | uppercase }}
```

Output:

```text
RAHUL
```

---

### 2. LowerCasePipe

```html
{{ name | lowercase }}
```

Output:

```text
rahul
```

---

### 3. TitleCasePipe

```html
{{ name | titlecase }}
```

Output:

```text
Rahul Shinde
```

---

### 4. DatePipe

```html
{{ today | date }}
```

Output:

```text
06-Jun-2026
```

---

### 5. CurrencyPipe

```html
{{ amount | currency:'INR' }}
```

Output:

```text
₹1000
```

---

### 6. PercentPipe

```html
{{ 0.85 | percent }}
```

Output:

```text
85%
```

---

## Exam Answer (5 Marks)

Pipes are used to transform and format data in Angular templates. Angular provides several built-in pipes such as:

1. UpperCasePipe – converts text to uppercase.
2. LowerCasePipe – converts text to lowercase.
3. TitleCasePipe – converts first letter of each word to uppercase.
4. DatePipe – formats dates.
5. CurrencyPipe – formats currency values.
6. PercentPipe – formats numbers as percentages.

Pipes improve readability and presentation of data.

# Q2(b) Attribute Binding

Ye exam me bahut baar aata hai.

---

## Problem

HTML me attribute hote hain.

Example:

```html
<img src="photo.jpg">
```

Yaha `src` ek attribute hai.

Angular me agar value dynamic ho to?

Attribute Binding use karte hain.

---

## Syntax

```html
[attr.attribute-name]="value"
```

---

## Example

```html
<button [attr.disabled]="isDisabled">
Click Me
</button>
```

Agar:

```ts
isDisabled = true
```

To button disable ho jayega.

---

## Another Example

```html
<td [attr.colspan]="2">
Employee Data
</td>
```

---

## Exam Answer

Attribute Binding is used to bind HTML attributes dynamically in Angular. It is represented using the syntax:

```html
[attr.attribute-name]="expression"
```

It allows Angular to update HTML attributes based on component data.

Example:

```html
<button [attr.disabled]="isDisabled">
Submit
</button>
```

---

# Difference (Exam Trick)

Bahut students confuse hote hain.

### Property Binding

```html
[disabled]="true"
```

DOM property ko bind karta hai.

### Attribute Binding

```html
[attr.disabled]="true"
```

HTML attribute ko bind karta hai.

---

# Recall

Attribute Binding syntax?

Answer:

```html
[attr.attribute-name]="value"
```

# Q2(c) Lazy Loading vs Eager Loading

Ye theory question pakka lag raha hai.

---

## Eager Loading

Application start hote hi saare modules load.

```text
App Start
↓
Module A
Module B
Module C
Sab Load
```

Advantages:

* Fast navigation

Disadvantages:

* Initial loading slow

---

## Lazy Loading

Sirf jab user page open kare tab module load.

```text
App Start
↓
Home Module Load

User Profile Page Open
↓
Profile Module Load
```

Advantages:

* Fast startup
* Better performance

Disadvantages:

* First time page open karne me thoda delay

---

## Exam Table

| Feature             | Eager Loading | Lazy Loading |
| ------------------- | ------------- | ------------ |
| Loading Time        | Startup par   | Need par     |
| Initial Performance | Slow          | Fast         |
| Memory Usage        | More          | Less         |
| Application Size    | Large         | Small        |
| Best For            | Small Apps    | Large Apps   |

---

## Exam Answer

Eager Loading loads all modules at application startup, whereas Lazy Loading loads modules only when required. Lazy Loading improves application performance and reduces initial load time, making it suitable for large applications.
